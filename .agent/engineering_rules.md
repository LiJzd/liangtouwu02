# 🔒 两头乌项目 - 工程律令与安全规约 (Engineering Rules & Security)

> **注意**：本文件是 AI 开发、数据读写、前端渲染以及多租户交互时的最高安全红线。

---

## 1. 启动与路径定位规约 (Startup & Absolute Path)
*   **根目录唯一启动原则**：始终从对应的子项目根目录启动应用（使用 `python run.py` 或 `python -m src...`），**严禁在深层子目录直接运行脚本**，以避免 `ModuleNotFoundError`。
*   **路径绝对化定义**：在 Python 核心代码中，一律使用 `pathlib` 结合当前项目根目录的 `BASE_DIR` 定义路径，严禁硬编码相对路径或 Windows 宿主机的特定盘符（如 `C:\Users\lost\...`）。

---

## 2. 多租户物理级安全隔离 (Multi-Tenant Isolation)
*   **租户隔离红线**：后端与 AI 服务之间的全部 HTTP/Kafka 交互，以及前端提交的全部查询，**必须强制携带并校验 `X-User-ID` (或 `user_id` 参数)**。
*   所有 SQL 查询、Milvus 检索和本地暂存区读写，必须以租户 ID 为强过滤条件，严格执行租户级数据物理隔离，严禁越权。

---

## 3. LLM 容错与空解析物理级联回滚 (AI Parse Guard)
为了彻底防护大模型“幻觉”及输出格式不规则导致的系统卡顿或垃圾数据堆积，系统必须挂载以下双层防御：

### 🧼 结构化 JSON 输入清洗器
在解析 LLM 输出前，必须前置增加 `cleanJSON` 解析器，自动识别并安全剥离 Markdown 的 ` ```json ` 物理标签，保障 `json.loads` 的绝对稳定性：
```python
def clean_json_string(raw_text: str) -> str:
    # 安全剥离 markdown 标签，获取纯净 JSON 串
    ...
```

### 🗑️ 空解析物理级联回滚 (State Rollback)
当 LLM 解析活动长度为 0 或由于校验失败判定为无效活动时，必须触发彻底清理逻辑：
1. **物理删除**：立即物理删除磁盘上该任务已写入的所有暂存文件。
2. **事务级回退**：使用数据库事务级联物理删除/回退对应的表记录（例如已生成的空 `uploaded_files` 库记录）。
3. **安全响应**：向前端返回 HTTP 400 指示错误，杜绝前端堆积占位垃圾卡片。

### 🧠 思考链过滤隔离
大模型的思考链 (Reasoning chain) 仅用于后台审计，在返回给前端或最终用户前，**必须在网络传输协议层进行物理过滤与剥离**，防止技术实现细节外泄或污染最终回复。

---

## 4. 前端性能与 XSS 深度防御 (Frontend Optimization & XSS)
*   **富文本 XSS 深度防御**：前端渲染 AI 生成的富文本或 Markdown 时，**必须且只能信任成熟的 DOMPurify 库进行物理清洗**，严禁使用自定义正则替换 `<script>` 标签，防止 `iframe`、`object` 及 SVG 事件注入。
*   **动态长列表渲染优化**：在养殖场监控等长列表（如 500+ 生猪卡片）渲染中，**严禁应用重度 CSS 滤镜（如高斯模糊 `backdrop-filter: blur()`）**。一致改用高性能的 HSL/RGBA 透明度背景，并引入 `content-visibility: auto` 以极大降低 GPU 重绘开销，防御 Chrome 进程崩溃。
*   **高频操作防抖**：搜索输入框（Search Input）及物联网设备轮询等高频 UI 事件，必须强制挂载 `debounce` (防抖) 逻辑，防止主线程过载死锁。

---

## 5. 并发与数据库原子性保障 (Database Atomicity)
*   **事务锁定红线**：多智能体交互的历史记录、设备状态更新等关键写操作，**必须强制包裹在数据库事务 (`BEGIN` / `COMMIT`) 中**。
*   **禁止非原子性操作**：在高并发场景下，严禁使用非原子性的“先删除、后插入”覆盖逻辑，避免高并发读写交错时产生短暂的“查询空窗期”数据漏洞。
